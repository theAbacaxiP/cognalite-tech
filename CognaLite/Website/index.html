    Introducing Swivel...

    One of our goals is to make technology accessible to everyone, so what better way to start than with the next generation of experts.

    Here to revolutionise the video teaching format, we are working on a prototype of swivel- a wall mounted phone holder that uses image processing to orient itself to face its user or to lock it in a fixed position. By utilising image processing in order to deliver a more interractive way to make videos. We took the familiar, basic car phone holder design and extrapolated it to provide a full range of motion as well as control of the distance between the camera and the wall.

    By using only 4 joints (as opposed to multiple universal joints in sequence) to produce a full range of motion and orientation, we can keep the price low.

    To simplify the design, an app will be developed in order to control swivel. It uses the phone's camera to capture images passively, then detecting faces and/or objects in order to calculate the required motion of swivel.

    Alternatively, you can fix swivel in one place. Using the phone's microphone (optional), you will be able to command swivel to "lock" or "unlock" its position.

    By utilising these features we deliver an easy-to-use tool that is simplistic and familiar; user friendly in all regards and all with an optimised price-point.

    Our Design Process:

    Our design process took off from a small paper and foam model which was used to get a sense of scale of the project

    The hinges for our works-like prototype are designed using servo motors (small, powerful motors which have a high level of control) and connected using pvc tubes.

    The electronics all operate under arduino (a microcontroller board) which allows for the motors to be controlled by a pc

    Our next step is the integration of the web app. A web app will be created using python's OpenCV library (image processing) and transmitted via a bluetooth chip extension to the arduino

    The reason why we use arduino is because it is operated on an open source software, meaning everyone can edit the code and change the way the motors move to their liking, providing more customisability and utility. If the code ever gets broken, then a self restore tool will be available on the website (up and coming)- redownloading the code and uploading it to the arduino.

    One way we can make swivel more interractive is to create a custom interface through the swivel app (up and coming) which can adjust to the user by adding custom verbal commands, something which can be easilly adjusted for with python using the pocketsphinx (speech recognition) module.

    Swivel's Competitors:

    Our main competitor is the iJoy Chase (a desk mounted phone holder that tracks the user's position). The iJoy Chase is almost the same as swivel in that it uses image processing via an app on the user's phone however the swivel allows for greater range of motion and customisability.

Pros:

Swivel offers a wider range of motion

Swivel offers more customisation

Swivel can be opperated using voice commands

Cons:

Swivel is more expensive than the iJoy

Swivel is more fragile due to the joints

Swivel requires either wall mounting or clamp mounting
